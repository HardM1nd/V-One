# Функционал маршрутов полетов

## Что добавлено

### Backend

1. **Модель FlightRoute** (`backend/post/models.py`):
   - Название маршрута
   - Точка отправления и назначения
   - Описание
   - Дата полета
   - Длительность полета
   - Расстояние
   - Тип самолета
   - Файл маршрута (опционально)
   - Публичный/приватный маршрут
   - Лайки и сохранения

2. **API Endpoints** (`backend/post/api/route_views.py`):
   - `GET /api/post/routes/` - список всех публичных маршрутов
   - `POST /api/post/routes/create/` - создание маршрута
   - `GET /api/post/routes/<id>/` - детали маршрута
   - `PUT /api/post/routes/<id>/update/` - обновление маршрута
   - `DELETE /api/post/routes/<id>/delete/` - удаление маршрута
   - `POST /api/post/routes/<id>/like/` - лайк/анлайк
   - `POST /api/post/routes/<id>/save/` - сохранение/удаление из сохраненных
   - `GET /api/post/routes/my/` - мои маршруты
   - `GET /api/post/routes/saved/` - сохраненные маршруты

3. **Фильтрация и сортировка**:
   - По пилоту
   - По типу самолета
   - По дате создания/полета
   - По расстоянию

### Frontend

1. **Компоненты**:
   - `RouteCard` - карточка маршрута
   - `RouteList` - список маршрутов с фильтрами
   - `RouteForm` - форма создания/редактирования
   - `RouteDetail` - детальная страница маршрута
   - `Routes` - главная страница с табами

2. **Интеграция**:
   - Раздел "Routes" в навигации
   - Вкладка "Routes" в профиле пилота
   - Маршруты отображаются в профиле

3. **Функционал**:
   - Создание маршрутов
   - Просмотр всех/своих/сохраненных маршрутов
   - Лайки и сохранения
   - Редактирование и удаление своих маршрутов
   - Фильтрация и сортировка

## Использование

1. Перейдите в раздел "Routes" в меню
2. Создайте новый маршрут через вкладку "Создать"
3. Заполните информацию о маршруте
4. Просматривайте маршруты других пилотов
5. Сохраняйте понравившиеся маршруты
6. В профиле во вкладке "Routes" видны ваши маршруты

## Миграции

При запуске через Docker миграции создадутся автоматически:

```bash
docker-compose exec backend python manage.py makemigrations post
docker-compose exec backend python manage.py migrate
```
